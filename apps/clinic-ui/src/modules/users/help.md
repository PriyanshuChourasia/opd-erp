# Users — User Directory

## What is this page?

The Users page (`/organisation/users`) is a read-only directory of all user accounts in the system. It displays user names, emails, roles, and account status. User management (creation, editing) is handled through the Auth registration flow and the Doctors module's "create with user" feature.

## What actions can be done?

- **View users** — Browse the paginated list of all registered users.
- **Search** — Search users by name or email address.

## What features does it hold?

- **Paginated DataTable** — Avatar/initials, full name, email, role badge, active/inactive status, and creation date.
- **Role badges** — Color-coded badges showing each user's assigned role (Super Admin, Receptionist, Doctor, Assistant).
- **Avatar initials** — Generated from the user's first and last name.
- **Active/inactive status** — Visual indicator of account status.
- **Read-only** — Users are created through registration, doctor creation, or seed data; this page is for viewing only.
